# Static Files

`express.static` is the built-in middleware for serving files from disk —
HTML, CSS, JS, images, anything. It maps URL paths to filesystem paths and
streams the bytes to the client.

## The basic shape

```js
import express from 'express';
import path from 'node:path';

const app = express();
app.use(express.static(path.join(import.meta.dirname, 'public')));
```

Now a request for `GET /logo.png` looks in `public/logo.png` and serves it
if it exists. If not, the middleware calls `next()` and the request
continues down the stack.

## Mounting under a prefix

```js
app.use('/assets', express.static('public'));
```

`GET /assets/logo.png` → `public/logo.png`. The `/assets` prefix is
**stripped** before the static middleware looks for the file.

## What it sets automatically

For each file:

- **`Content-Type`** based on extension (`.png` → `image/png`).
- **`Content-Length`** from file size.
- **`Last-Modified`** from the file mtime.
- **`ETag`** from inode + size + mtime.
- Handles `If-Modified-Since` and `If-None-Match` — sends `304 Not
  Modified` when the client's cached copy is current. Big bandwidth win.

You don't need to do any of this manually — that's the point of the
middleware.

## Cache-Control

Set how long browsers/CDNs cache files:

```js
app.use(express.static('public', {
  maxAge: '7d',           // → Cache-Control: max-age=604800
  immutable: true,        // → Cache-Control: immutable (for hashed filenames)
}));
```

For app bundles with content-hashed filenames (`app.abc123.js`), use
`maxAge: '1y'` plus `immutable: true` — the browser caches them forever
and only re-fetches when the hash changes.

## What gets exposed

The middleware serves anything in the directory. That's both the feature
and the foot-gun.

<Callout variant="warn">
**Don't `express.static` the project root** or any folder containing
secrets:

```js
app.use(express.static('./')); // exposes .env, package.json, etc
```

Use a dedicated `public/` folder with nothing sensitive in it. The
middleware also blocks `..` traversal by default, so you can't escape the
configured directory, but you can still expose everything inside it.
</Callout>

## Index files

Requests for a directory serve `index.html` if it exists:

```
GET /docs/      → docs/index.html (if it exists)
```

Disable with `index: false`. Customize with `index: 'home.html'`.

## Dot-files

By default, dotfiles like `.env` or `.git/config` aren't served —
`express.static` ignores them. But you can override:

```js
express.static('public', { dotfiles: 'allow' });  // dangerous
```

There's almost never a reason to. Leaving the default (`ignore`) is the
safe choice.

## Where in the chain

Mount static **late** if you have API routes — otherwise a request for
`/api/users` might be checked against `public/api/users` first. In
practice the static middleware just calls `next()` when nothing matches,
so order doesn't change correctness, but it can change performance
(filesystem lookups have a cost).

Common pattern:

```js
app.use(express.json());
app.use('/api', apiRouter);
app.use(express.static('public'));   // last
app.use((req, res) => res.status(404).send('Not Found'));
```

## When not to use it

For real production traffic, put the files behind a CDN or a reverse
proxy (nginx, Cloudflare) instead of Node. Reasons:

- Static file serving is a solved problem; nginx is dramatically faster
  at it than Node.
- A CDN puts files near the user.
- It frees the Node process for actual app logic.

`express.static` is great for development, small apps, and single-page-app
hosting (where you serve `index.html` for unknown routes). Beyond that,
move it out of Node.
