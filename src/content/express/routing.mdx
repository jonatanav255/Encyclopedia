# Routing

Routing is how Express decides which handler runs for an incoming request. A
route is defined by an **HTTP method** plus a **path pattern**, and the first
route that matches both wins.

## Basic shape

```js
app.get('/users', (req, res) => res.json(users));
app.post('/users', (req, res) => res.status(201).json(create(req.body)));
app.get('/users/:id', (req, res) => res.json(find(req.params.id)));
```

## Path parameters

A segment that starts with `:` becomes a named parameter, available on
`req.params`:

```js
app.get('/users/:id/posts/:postId', (req, res) => {
  // req.params.id      → "42"
  // req.params.postId  → "7"
});
```

A trailing `*` matches the rest of the path and lands on `req.params[0]`:

```js
app.get('/files/*', (req, res) => {
  // GET /files/img/cat.png  →  req.params[0] === "img/cat.png"
});
```

## Order matters here too

Express walks the routes top-to-bottom and stops at the **first** match. That
means a broader route registered first will swallow more specific ones
registered later:

```js
app.get('/users/:id', handlerA);   // wins for /users/me
app.get('/users/me', handlerB);    // never reached
```

<Callout variant="warn">
Put specific routes **before** parameterized ones. If `/users/me` and
`/users/:id` both exist, `/users/me` must come first.
</Callout>

## Routers

For anything bigger than a toy app, split routes into `express.Router()`
instances and mount them:

```js
const usersRouter = express.Router();
usersRouter.get('/', listUsers);
usersRouter.get('/:id', getUser);

app.use('/users', usersRouter);
```

Each router is just a mini-app with its own middleware stack — same rules,
same ordering, just scoped.
