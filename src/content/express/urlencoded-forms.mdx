# URL-encoded Forms

Plain HTML `<form>` submissions don't send JSON. They send
`application/x-www-form-urlencoded` — the same format a query string uses,
but in the request body. `express.json()` ignores it. You need
`express.urlencoded()`.

## The setup

```js
import express from 'express';
const app = express();

app.use(express.urlencoded({ extended: true }));

app.post('/login', (req, res) => {
  console.log(req.body); // { email: '...', password: '...' }
});
```

When the form looks like this:

```html
<form action="/login" method="POST">
  <input name="email" />
  <input name="password" type="password" />
  <button>Sign in</button>
</form>
```

The browser sends:

```
POST /login HTTP/1.1
Content-Type: application/x-www-form-urlencoded

email=jon%40example.com&password=hunter2
```

`express.urlencoded()` decodes the percent-encoding and splits the
key/value pairs into `req.body`.

## `extended: true` vs `extended: false`

The `extended` option controls the parser:

- `extended: false` — uses the built-in `querystring` module. Flat
  key/value pairs only. `?a=1&b=2` → `{ a: '1', b: '2' }`.
- `extended: true` — uses the `qs` library. Supports nested objects and
  arrays: `?user[name]=Jon&tags=js&tags=ts` →
  `{ user: { name: 'Jon' }, tags: ['js', 'ts'] }`.

The Express docs nudge you toward `extended: true` for API ergonomics, but
honestly: most HTML forms don't need nesting, and the `qs` parser has had
denial-of-service CVEs historically (deeply nested input → expensive parse).
For plain `<form>`s, `extended: false` is fine and slightly safer.

## Use it alongside `express.json()`

You can mount both. They look at `Content-Type` and only run for matching
requests.

```js
app.use(express.json());
app.use(express.urlencoded({ extended: false }));
```

Now JSON API clients (`Content-Type: application/json`) and HTML form
submissions (`Content-Type: application/x-www-form-urlencoded`) both end up
on `req.body`.

## What it can't handle

- **File uploads** — those use `multipart/form-data`, not urlencoded.
  Reach for `multer`. See **File Uploads**.
- **Anything bigger than the limit** — default is 100kb. Configure with
  `express.urlencoded({ limit: '500kb' })`.

<Callout variant="tip">
If you're building a JSON-only API (no HTML forms), you don't need
`express.urlencoded()`. Skip it. Every parser you register is a small
attack surface — only mount the ones you actually use.
</Callout>

## The values are strings

Just like params and query — even numeric form fields arrive as strings.
`req.body.age === '42'`, not `42`. Validate and coerce before using.
