# Equality

JavaScript has three equality operators. Two of them are useful.

## `===` strict equality

Same type, same value. No coercion.

```js
1 === 1;          // true
'a' === 'a';      // true
1 === '1';        // false   — different types
null === null;    // true
undefined === undefined; // true
NaN === NaN;      // false   <-- the one exception
{} === {};        // false   — different objects
```

`NaN !== NaN` because IEEE 754 says so. To test for `NaN` use
`Number.isNaN(x)` (or check `x !== x`).

`{} !== {}` because object equality is **by reference**, not by
structure. Two empty objects are two different objects.

## `==` loose equality

Coerces both sides until they're the same type, then compares. The
rules are long and surprising; here are the practical ones:

```js
1 == '1';         // true   (string → number)
true == 1;        // true   (boolean → number)
false == 0;       // true
null == undefined; // true
null == 0;        // false  <-- null only equals undefined and null
'' == 0;          // true   ('' → 0)
[] == false;      // true   ([] → '' → 0)
'1,2' == [1, 2];  // true   (array toString)
```

The `null == undefined` rule is the **one** useful behavior — it lets
you check "is this nullish?" in one comparison:

```js
if (x == null) { /* either null or undefined */ }
```

Everything else about `==` is a footgun.

## `Object.is`

Almost identical to `===`, with two exceptions:

```js
Object.is(NaN, NaN);   // true   (=== returns false)
Object.is(0, -0);      // false  (=== returns true)
Object.is(1, 1);       // true   (same as ===)
```

Use `Object.is` when you specifically need `NaN === NaN` or want to
distinguish `0` from `-0`. For everything else, `===`.

## Reference equality

Objects are equal only if they're the same object:

```js
const a = { x: 1 };
const b = { x: 1 };
const c = a;

a === b;   // false — different objects
a === c;   // true  — same reference
```

To compare object contents you need a deep-equal helper or compare
specific fields. Common deep-equal libraries: `lodash.isequal`,
`fast-deep-equal`. Or `JSON.stringify(a) === JSON.stringify(b)` for a
crude but often-good-enough check (fails on `undefined`, functions,
Dates, Maps, and key order).

## The rule

- Use `===` for all comparisons.
- Use `x == null` only for the specific "nullish check" idiom.
- Use `Number.isNaN(x)` to detect `NaN`.
- Never write `== false`, `== true`, `== 0`, `== ''` — those are bugs
  waiting to happen.

ESLint's `eqeqeq` rule catches stray `==` for you. Turn it on.
