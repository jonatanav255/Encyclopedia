# Arrays

Arrays in JavaScript are ordered, indexed, dynamically sized lists.
They're also just objects under the hood — the indices are string keys
and `.length` is a regular property — but they behave like arrays for
practical purposes.

## Creating arrays

```js
const a = [];                    // empty
const b = [1, 2, 3];             // literal
const c = Array.from('abc');     // ['a', 'b', 'c'] — from iterable
const d = Array.from({ length: 3 }, (_, i) => i * 2); // [0, 2, 4]
const e = Array.of(7);           // [7]   — single element
const f = new Array(7);          // length 7, all empty slots — avoid
```

`new Array(7)` creates a "sparse" array of length 7 with no actual
elements — a common footgun. Use `Array.from({ length: 7 })` or
`Array(7).fill(0)` instead.

## Mutation vs non-mutation

Half the array methods mutate the array; the other half return a new
one. Knowing which is which prevents bugs.

**Mutating** (modify in place):
`push`, `pop`, `shift`, `unshift`, `splice`, `sort`, `reverse`, `fill`.

**Non-mutating** (return new):
`concat`, `slice`, `map`, `filter`, `reduce`, `flat`, `flatMap`,
`indexOf`, `includes`, `join`, `find`, `findIndex`, `some`, `every`.

```js
const xs = [3, 1, 2];

xs.sort();              // mutates xs to [1, 2, 3], returns it too
const ys = [...xs].sort(); // non-mutating: clone, then sort
```

ES2023 added immutable counterparts: `toSorted`, `toReversed`,
`toSpliced`, `with`. Use them when you want functional semantics:

```js
const xs = [3, 1, 2];
const sorted = xs.toSorted(); // new array; xs unchanged
```

## Iteration

Three core patterns:

```js
const xs = [10, 20, 30];

for (const x of xs) console.log(x);          // each value
for (const [i, x] of xs.entries()) console.log(i, x); // index + value
xs.forEach((x, i) => console.log(i, x));     // callback form
```

`for...of` is the modern default. `forEach` doesn't support `break` or
`return` (it ignores them), which is occasionally an issue. `for...in`
is for **objects**, not arrays — it iterates property names including
inherited ones; never use it on arrays.

```js
const ys = [1, 2, 3];
ys.extra = 'oops';

for (const i in ys) console.log(i);   // "0", "1", "2", "extra"  <-- bad
for (const v of ys) console.log(v);   // 1, 2, 3
```

## `map` / `filter` / `reduce`

The three workhorses.

```js
const nums = [1, 2, 3, 4];

nums.map(n => n * 2);                  // [2, 4, 6, 8]
nums.filter(n => n % 2 === 0);         // [2, 4]
nums.reduce((sum, n) => sum + n, 0);   // 10
```

`reduce` is the most general — it can express `map` and `filter` too,
plus anything that builds up a value:

```js
// Build a frequency map
const words = ['a', 'b', 'a', 'c', 'b', 'a'];
const counts = words.reduce((acc, w) => {
  acc[w] = (acc[w] ?? 0) + 1;
  return acc;
}, {});
// { a: 3, b: 2, c: 1 }
```

For a fluent chain:

```js
orders
  .filter(o => o.status === 'shipped')
  .map(o => o.total)
  .reduce((a, b) => a + b, 0);
```

## Common gotchas

### `sort` is lexicographic by default

```js
[10, 2, 1].sort();          // [1, 10, 2]   <-- not what you want
[10, 2, 1].sort((a, b) => a - b); // [1, 2, 10]
```

Always pass a comparator for numbers.

### `splice` vs `slice`

`slice(start, end)` returns a new array (non-mutating).
`splice(start, deleteCount, ...items)` mutates and returns the removed
elements.

```js
const xs = [1, 2, 3, 4];
xs.slice(1, 3);    // [2, 3] — xs unchanged
xs.splice(1, 2);   // [2, 3] — xs is now [1, 4]
```

The names are confusingly close. Mistake one for the other and you
mutate state you didn't mean to.

### Holes

Arrays can have "holes" — slots that are neither set nor `undefined`:

```js
const a = [1, , 3];
a.length;          // 3
a[1];              // undefined
a.map(x => x);     // [1, <empty>, 3]   — map skips the hole
a.filter(x => true); // [1, 3]          — filter removes it
```

Holes propagate through some methods and not others. Avoid creating
sparse arrays.

### Equality

`[1] !== [1]`. Always. Different objects, different references.

## Spread and rest

```js
const a = [1, 2, 3];
const b = [...a, 4];      // [1, 2, 3, 4] — spread copy
const [first, ...rest] = a; // first = 1, rest = [2, 3]

function sum(...nums) {     // rest in parameters
  return nums.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3); // 6
```

Spread is the idiomatic shallow copy. For deep, use `structuredClone`.

## When to reach for what

- **Pure transforms** → `map` / `filter` / `reduce` / `flatMap`.
- **Searching** → `find` / `findIndex` / `includes`.
- **Mutation needed** → `push` / `splice`. Prefer the immutable form
  if the array is shared.
- **Lookup by key** → use a `Map` or plain object, not an array
  search. Array search is O(n).
- **Unique elements** → `new Set(arr)` + `[...set]`.
