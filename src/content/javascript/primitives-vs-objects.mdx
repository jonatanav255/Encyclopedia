# Primitives vs objects

Every value in JavaScript is either a **primitive** or an **object**.
The difference shapes how assignment, comparison, and function arguments
behave.

## The two categories

**Primitives** (immutable, copied by value):
`string`, `number`, `bigint`, `boolean`, `undefined`, `null`, `symbol`.

**Objects** (mutable, passed by reference):
plain objects `{}`, arrays `[]`, functions, dates, regexes, Maps, Sets,
and every class instance.

```js
typeof 'hi';       // "string"     primitive
typeof 1;          // "number"     primitive
typeof true;       // "boolean"    primitive
typeof undefined;  // "undefined"  primitive
typeof null;       // "object"     primitive (typeof bug)
typeof Symbol();   // "symbol"     primitive

typeof {};         // "object"
typeof [];         // "object"
typeof (() => {}); // "function"   (technically callable object)
```

## Copy by value vs copy by reference

```js
let a = 5;
let b = a;
b = 10;
console.log(a); // 5 — primitive copied

let x = { count: 5 };
let y = x;
y.count = 10;
console.log(x.count); // 10 — same object
```

For primitives the value is duplicated; the two variables are
independent. For objects the **reference** is duplicated — both
variables point to the same object.

## Function arguments

Same rule. Primitives are copied; objects are passed by reference.

```js
function bumpPrim(n) { n = n + 1; }
function bumpObj(o)  { o.count = o.count + 1; }

let n = 1;
bumpPrim(n);
console.log(n); // 1 — unchanged

let o = { count: 1 };
bumpObj(o);
console.log(o.count); // 2 — mutated
```

But reassigning the parameter doesn't affect the caller — `o = {}`
inside the function would just rebind the local `o`, not change what
the caller's `o` points to.

```js
function replace(o) { o = { count: 99 }; }
let o = { count: 1 };
replace(o);
console.log(o.count); // 1 — caller's o still points to the original
```

This is what "JS is pass by value" really means: it passes references
*by value*. The reference is copied; what it points to is shared.

## Immutability of primitives

You can't change a primitive — only reassign the variable:

```js
let s = 'hello';
s[0] = 'H';      // silently does nothing (strict mode throws)
console.log(s);  // "hello"

s = s.toUpperCase();
console.log(s);  // "HELLO" — the variable was reassigned to a new string
```

Methods on primitives like `'hello'.toUpperCase()` return *new* values.
The original is untouched.

## Equality

Primitives compare by value; objects compare by reference.

```js
'abc' === 'abc';        // true
{ a: 1 } === { a: 1 };  // false — different objects
[1, 2] === [1, 2];      // false — different arrays
```

To compare object contents you need deep equality (a library, or a
manual structural check).

## Boxing

When you call a method on a primitive, the engine **temporarily wraps**
it in an object (`String`, `Number`, `Boolean`) so it has methods.

```js
'hi'.toUpperCase(); // wraps 'hi' in a String object, calls method, discards wrapper
```

This is invisible to you. The wrapper exists for the duration of the
expression, then is GC'd. Don't ever use the explicit constructors:

```js
const s = new String('hi'); // object — typeof is "object", not "string"
s === 'hi';                 // false — different types
if (new Boolean(false)) console.log('huh');  // logs! Boolean(false) is truthy
```

`new String`, `new Number`, `new Boolean` are footguns. Use the literals
or `String(x)` / `Number(x)` (without `new`) for conversion.

## When the distinction matters

- **Caching by identity** — `Map` keys, `Set` membership, `===` checks
  rely on reference equality for objects.
- **State management** — React, Redux, and friends use reference
  equality to know if state changed. Mutating an object in place won't
  trigger a re-render; returning a new object will.
- **Cloning** — copying a primitive is free; cloning an object is
  not. Shallow (`{ ...obj }`) vs deep (`structuredClone(obj)`) matters
  for nested structures.
- **Default function args** — a default `[]` is a new array each call,
  but a default at module scope is shared across all calls.
