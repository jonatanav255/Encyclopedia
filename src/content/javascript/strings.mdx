# Strings

Strings in JavaScript are immutable sequences of UTF-16 code units.
"Immutable" means every "modification" actually returns a new string.

## Creating strings

```js
const a = 'hello';
const b = "hello";        // single or double — pick one and lint for it
const c = `hello`;        // backtick — template literal
const d = String(42);     // "42" — explicit conversion
```

Don't use `new String('hi')` — that creates a wrapper *object*, not a
string primitive (`typeof` is `"object"`, comparisons fail).

## Template literals

Backtick strings support interpolation and multi-line text:

```js
const name = 'jon';
const greeting = `hi ${name}!`;
const multi = `line 1
line 2
line 3`;
```

Inside `${}` you can use any expression — `${user.firstName}`,
`${1 + 2}`, `${items.length}`. They beat `'hi ' + name + '!'` for
readability.

### Tagged templates

A function placed in front of a template literal receives the strings
and interpolations separately:

```js
function highlight(strings, ...values) {
  return strings.reduce((acc, s, i) =>
    acc + s + (values[i] !== undefined ? `<b>${values[i]}</b>` : ''), '');
}

const name = 'jon';
highlight`Hi ${name}, welcome.`;
// "Hi <b>jon</b>, welcome."
```

Used by `styled-components`, GraphQL clients, SQL libraries, i18n
tools. The tag function can validate, escape, or transform.

## Common methods

```js
'hello'.length;             // 5
'hello'.toUpperCase();      // "HELLO"
'  hi  '.trim();            // "hi"
'  hi  '.trimStart();       // "hi  "
'a,b,c'.split(',');         // ['a', 'b', 'c']
['a', 'b'].join('-');       // "a-b"
'hello'.slice(1, 4);        // "ell"
'hello'.substring(1, 4);    // "ell"  (subtly different — avoid)
'hello'.includes('ell');    // true
'hello'.startsWith('he');   // true
'hello'.endsWith('lo');     // true
'hello'.indexOf('l');       // 2  (first match) — -1 if missing
'hello'.replace('l', 'L');  // "heLlo"  — only first match
'hello'.replaceAll('l', 'L'); // "heLLo"
'5'.padStart(3, '0');       // "005"
'a'.repeat(3);              // "aaa"
```

### `slice` vs `substring` vs `substr`

- `slice(start, end)` — the modern choice. Supports negative indices
  (counted from the end).
- `substring(start, end)` — swaps args if start > end, doesn't support
  negatives. Quirky.
- `substr(start, length)` — deprecated.

Use `slice`. Forget the others exist.

### `replace` vs `replaceAll`

`replace` only replaces the **first** match with a string argument.
Use a regex with `/g` flag, or `replaceAll`, for global replacement.

```js
'a.b.c'.replace('.', '-');       // "a-b.c"
'a.b.c'.replace(/\./g, '-');     // "a-b-c"
'a.b.c'.replaceAll('.', '-');    // "a-b-c"
```

## Iteration

Strings are iterable. `for...of` walks **code points** (handles
surrogate pairs correctly):

```js
const s = 'hi👋';
for (const c of s) console.log(c);
// "h", "i", "👋"   (👋 is one code point, two code units)

s.length;             // 4 — code units, not code points
[...s].length;        // 3 — spread iterates code points
```

`s[i]` accesses **code units**, not characters. For emoji and
non-BMP characters, that means broken halves of a character:

```js
const s = '👋';
s[0];   // "\uD83D"   first half of the surrogate pair
s.length; // 2

// To get the actual code point:
[...s][0]; // "👋"
```

## Comparing strings

```js
'abc' === 'abc';        // true — value comparison
'abc' < 'abd';          // true — lexicographic
'10' < '9';             // true — string comparison! ('1' < '9')
```

For localized comparison (different alphabets, accents, case), use
`Intl.Collator`:

```js
const cmp = new Intl.Collator('en', { sensitivity: 'base' }).compare;
cmp('Ångström', 'angstrom'); // 0 — treated as equal
```

## Encoding pitfalls

JavaScript strings are UTF-16 internally. Be careful when:

- **Counting "characters"** — use `[...str].length` for code points.
- **Reversing** — `str.split('').reverse().join('')` breaks emoji.
  Use `[...str].reverse().join('')`.
- **Slicing** — `str.slice(0, n)` might cut a surrogate pair in half.
- **Encoding to bytes** — use `TextEncoder` (UTF-8 by default), not
  `Buffer.from(str)` if you're in a browser.

## Common patterns

### Build a string from parts

```js
const parts = ['hello', 'world'];
parts.join(' ');           // "hello world" — clearest

// avoid string concatenation in a loop
let out = '';
for (const p of parts) out += p; // OK for small N, slow for large
```

### Format numbers and dates

Use `Intl.NumberFormat` / `Intl.DateTimeFormat`, not manual building:

```js
new Intl.NumberFormat('en-US', { style: 'currency', currency: 'USD' })
  .format(1234.56);          // "$1,234.56"

new Intl.DateTimeFormat('en-GB').format(new Date());
// "20/05/2026"
```

### Escape user input for HTML

Never inject user strings directly into HTML. Use a templating engine
or React's automatic escaping. Manual escaping is for the four
characters `& < > "` — but DOM-based insertion (`textContent`,
`createTextNode`) is safer.

## The mental model

Strings are values, not buffers. Every operation returns a new string;
the original is untouched. They're cheap to create thanks to V8's
internal sharing, but a tight loop building strings still allocates a
lot — drop into an array + `join` if it matters.
