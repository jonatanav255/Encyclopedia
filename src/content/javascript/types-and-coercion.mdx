# Types and coercion

JavaScript has **eight types**: seven primitives (`string`, `number`,
`bigint`, `boolean`, `null`, `undefined`, `symbol`) and `object`
(everything else — arrays, functions, dates, regexes, plain objects).

```js
typeof 'hi';        // "string"
typeof 42;          // "number"
typeof 9007199254740993n; // "bigint"
typeof true;        // "boolean"
typeof undefined;   // "undefined"
typeof Symbol();    // "symbol"
typeof null;        // "object"   <-- historical bug, can't be fixed
typeof [];          // "object"
typeof function(){} // "function" <-- callable object
```

The two oddities — `typeof null === 'object'` and
`typeof function === 'function'` — are quirks you memorize. Everything
else is consistent.

## Primitives vs objects

Primitives are **immutable** and compared by value. Objects are
**mutable** and compared by reference.

```js
let a = 'hi';
let b = a;
b = 'bye';
console.log(a); // "hi" — string copied by value

let x = { name: 'jon' };
let y = x;
y.name = 'jen';
console.log(x.name); // "jen" — both point to the same object
```

This is *the* mental model you need for the rest of the language.

## Coercion

Coercion is the automatic conversion of one type to another. JS does it
in lots of places, sometimes helpfully, sometimes catastrophically.

### To string

`+` with any string operand coerces the other side to string:

```js
'1' + 1;        // "11"
'count: ' + 5;  // "count: 5"
[1, 2] + '';    // "1,2"  (array.toString joins with comma)
{} + '';         // "[object Object]"
```

### To number

Most other operators coerce both sides to number:

```js
'5' - 1;        // 4
'5' * '2';      // 10
'foo' - 1;      // NaN
true + 1;       // 2
null + 1;       // 1     (null → 0)
undefined + 1;  // NaN   (undefined → NaN)
```

The unary `+` is the canonical "make this a number" shorthand:

```js
+'5';      // 5
+'';        // 0
+'abc';     // NaN
+true;      // 1
+null;      // 0
+undefined; // NaN
```

### To boolean

In `if`, `&&`, `||`, `!`, and the ternary, values coerce to boolean.
The **falsy** values — anything that becomes `false` — are:

```js
false
0, -0, 0n
''         // empty string
null
undefined
NaN
```

Everything else is **truthy**, including `'0'`, `'false'`, `[]`, `{}`,
and `new Boolean(false)` (yes, really).

```js
if ([]) console.log('truthy');     // logs — empty array is truthy
if ('0') console.log('truthy');    // logs — non-empty string
if (0) console.log('truthy');      // doesn't log
```

## Why this matters

Coercion bites in three classic places:

```js
// 1. + means concat when one side is string
1 + '2'; // "12", not 3

// 2. checking "is this set" with truthiness misses 0 and ''
function setLevel(n) {
  if (!n) n = 1;   // BUG: setLevel(0) becomes setLevel(1)
}
// fix: if (n == null) n = 1;  or default parameter: function setLevel(n = 1)

// 3. JSON returns null for undefined values
JSON.stringify({ a: undefined, b: 1 }); // '{"b":1}'
```

## The rule of thumb

- Use `===` and `!==` everywhere. They never coerce.
- Use explicit conversions: `Number(x)`, `String(x)`, `Boolean(x)`.
- When checking "is this nullish?", use `x == null` (matches both
  `null` and `undefined`) or `x ?? default`.
- Don't rely on `+` to convert — use `Number()`.

Coercion is the reason JavaScript has a reputation for surprising
behavior. Once you internalize the falsy list and the string-vs-number
flip on `+`, the surprises mostly go away.
