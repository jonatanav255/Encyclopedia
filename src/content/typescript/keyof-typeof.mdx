# `keyof` and `typeof`

The two type-level operators you'll use most often after the basics.
`keyof` gives you the keys of a type; `typeof` (in a type position)
gives you the type of a value.

## `keyof`

```ts
type User = { id: number; name: string; email: string };

type Keys = keyof User;   // 'id' | 'name' | 'email'
```

`keyof T` is the union of T's keys (as string/symbol literals).

For objects with mixed key types:

```ts
type Mixed = { 1: 'a'; foo: 'b'; [sym]: 'c' };
type K = keyof Mixed;   // 1 | 'foo' | typeof sym
```

For arrays:

```ts
type ArrKeys = keyof string[];
// number | 'length' | 'push' | 'pop' | ... â€” all array methods
```

For maps:

```ts
type MapKeys = keyof Record<string, number>;
// string | number â€” Record includes a string index, plus numbers coerce
```

## `keyof` with generics

The classic typed-getter:

```ts
function get<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { id: 1, name: 'jon' };
get(user, 'name');    // string
get(user, 'missing'); // error
```

`K extends keyof T` constrains K to a real key. `T[K]` is the value
type at that key â€” see **Indexed access types** below.

## Indexed access types

```ts
type User = { id: number; name: string };

type Name = User['name'];          // string
type Either = User['id' | 'name']; // number | string
type Any = User[keyof User];        // number | string â€” all value types
```

`T[K]` accesses the type at key K. K can be a literal, a union, or
`keyof T` (which gives the union of all value types).

For arrays:

```ts
const arr = [1, 'a', true];
type ArrEl = (typeof arr)[number];  // number | string | boolean
```

`T[number]` is "the type of any element of T." Common pattern for
arrays.

For tuples:

```ts
type Pair = [string, number];
type First = Pair[0];     // string
type Second = Pair[1];    // number
type Any = Pair[number];  // string | number
```

## `typeof` (in a type position)

`typeof` in a type position refers to the type of a *value*:

```ts
const config = {
  port: 3000,
  host: 'localhost',
  debug: true,
};

type Config = typeof config;
// { port: number; host: string; debug: boolean }
```

Useful when:

- You have a config value and want to derive its type.
- A library exports a value but no type for it.
- You'd rather author the value and let the type follow.

```ts
const APP_VERSION = '1.2.3' as const;
type AppVersion = typeof APP_VERSION;   // '1.2.3'
```

Note: TypeScript's `typeof` (in *type* position) is unrelated to the
runtime `typeof` operator. Same word, different namespace.

## `typeof` + class

```ts
class Foo {}
type FooInstance = Foo;
type FooConstructor = typeof Foo;

const Ctor: typeof Foo = Foo;
new Ctor();
```

`typeof MyClass` is the constructor type. `MyClass` (used as a type)
is the instance type.

## Combining `keyof` and `typeof`

The fundamental "derive a union from an object literal" pattern:

```ts
const ROLES = {
  admin: 'Admin',
  user: 'User',
  guest: 'Guest',
} as const;

type RoleKey = keyof typeof ROLES;   // 'admin' | 'user' | 'guest'
type RoleLabel = (typeof ROLES)[RoleKey]; // 'Admin' | 'User' | 'Guest'
```

Why this is great: the source of truth is the ROLES object. Types
follow. Add a role to ROLES; the types update.

Without `as const`, the values widen to `string` and the second type
becomes `string`. `as const` is essential for this pattern.

## Real-world examples

### Derive a switch's payload type from a map

```ts
const handlers = {
  ADD: (n: number) => n + 1,
  SUB: (n: number) => n - 1,
  RESET: () => 0,
} as const;

type ActionType = keyof typeof handlers;   // 'ADD' | 'SUB' | 'RESET'
type Handler<T extends ActionType> = (typeof handlers)[T];
```

### Pick valid query params

```ts
const FILTERS = ['status', 'createdAt', 'updatedAt'] as const;
type Filter = typeof FILTERS[number];   // 'status' | 'createdAt' | 'updatedAt'

function setFilter(f: Filter, value: string) { ... }
```

### Type a config object's environment

```ts
const ENV = process.env;
type EnvVar = keyof typeof ENV;   // string (because process.env is typed as that)
```

When `ENV` is typed more strictly (e.g., via `dotenv` typings), this
becomes useful.

## `keyof` on generic types

```ts
function logProps<T>(obj: T): void {
  const keys = Object.keys(obj) as (keyof T)[];   // assertion
  for (const k of keys) console.log(k, obj[k]);
}
```

`Object.keys` returns `string[]` by default â€” it doesn't know about
your specific type's keys. The cast is the standard workaround.

`noUncheckedIndexedAccess` may make `obj[k]` `T[keyof T] | undefined`,
which is technically wrong but the compiler isn't smart enough.

## `keyof` and index signatures

```ts
type Dict = { [key: string]: number };
type DictKeys = keyof Dict;   // string | number
```

The `number` shows up because numeric keys are coerced to strings at
runtime â€” TS includes them in `keyof`. Slightly surprising; usually
not a problem.

```ts
type StringDict = Record<string, number>;
type K = keyof StringDict;    // string | number (same)
```

## Mapped types use `keyof`

```ts
type Readonly<T> = { readonly [K in keyof T]: T[K] };
type Partial<T> = { [K in keyof T]?: T[K] };
type Required<T> = { [K in keyof T]-?: T[K] };
```

These are the built-in utilities. Their definition is `keyof T`
iterated with `[K in keyof T]`. See **Mapped types**.

## Common pitfalls

### `keyof object` is `never`

```ts
type K = keyof object;   // never
```

`object` is the constraint "non-primitive" â€” it has no specific keys.
Use `keyof T` for a specific T, not `keyof object`.

### `keyof any` is broad

```ts
type K = keyof any;   // string | number | symbol
```

The union of all possible key types. Useful in `Record` constraints:
`type Record<K extends keyof any, V> = { [P in K]: V }`.

### Widening without `as const`

```ts
const PERMS = { read: 1, write: 2 };
type K = keyof typeof PERMS;   // string â€” keys widened!
```

Wait, no â€” keys of a regular object literal narrow correctly. It's
the *values* that widen:

```ts
type V = (typeof PERMS)[keyof typeof PERMS];   // number, not 1 | 2
```

If you want literal values, `as const`.

### Indexing with a wider key than expected

```ts
function f<K extends 'a' | 'b'>(key: K) {
  const obj = { a: 1, b: 'hi' };
  return obj[key];   // 1 | 'hi'  â€” the union of all value types
}
```

When you index with a union key, you get a union value. To get the
*specific* value type when the key narrows, you need conditionals.

## The discipline

- Use `keyof T` for "valid keys of T."
- Use `T[K]` for "value at key K."
- Use `typeof value` (in a type position) to derive types from
  runtime data.
- Use `as const` when you need literal types.
- Use the `keyof typeof` combination to derive unions from objects.

These four operators (`keyof`, `T[K]`, `typeof`, `as const`) cover
80% of "derive a type from a value or shape" needs. Master them
before reaching for conditional or mapped types.
